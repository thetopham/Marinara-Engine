Local development

Run and build the Tauri app from a clean checkout.

The app uses Vite for the React webview and Tauri v2 for the desktop shell. Tauri starts Vite on port 1420 during desktop development and bundles dist/ for production builds.

Prerequisites

Tool Why it is needed
Node.js and pnpm Install JavaScript dependencies and run Vite, TypeScript, and Tauri CLI scripts.
Rust stable toolchain Compile the Tauri host and Rust capability crates.
Platform Tauri prerequisites Install the OS webview/build prerequisites required by Tauri for your platform.

Install Dependencies

Run from the repo root:

pnpm install

Run The Desktop App

This is the normal local development command. It starts Vite through Tauri's beforeDevCommand, opens the Tauri window, and connects the webview to http://localhost:1420. 

pnpm tauri dev
Port 1420 is strict in vite.config.ts. If it is already in use, stop the process using that port before running Tauri dev.

Run The Web Shell Only

Use this when you only need the React webview. Tauri-only capabilities may not work correctly because the desktop host is not running. 

pnpm dev

Build The App

Run the frontend build:

pnpm build

Build the Tauri desktop bundle:

pnpm tauri build

Tauri runs pnpm build before bundling because src-tauri/tauri.conf.json sets beforeBuildCommand to pnpm build.

The public app name for bundles and the desktop window comes from src-tauri/tauri.conf.json. Keep productName and the main window title set to Marinara Engine.

Manual Pre-Alpha Platform Builds

Maintainers can run Pre-Alpha Platform Builds from GitHub Actions on the refactor branch. The workflow is manual-only, creates a draft GitHub release first, uploads artifacts, then publishes it as a pre-release with make_latest=false so it is not treated as the latest release.

Pre-alpha release notes intentionally warn that almost everything may be broken. These builds are unsigned, not connected to the stable updater, and should be tested only with throwaway data.

Run The Remote Rust Runtime

The remote runtime hosts the Rust API only. It does not serve the React UI. Desktop clients can point Settings > Advanced > Remote Runtime URL at this server to route supported storage, import, connection, model-list, and LLM calls through HTTP. 

cargo run --manifest-path src-tauri/Cargo.toml --bin marinara-server

By default it listens on:

http://127.0.0.1:8787

Check the server with:

curl http://127.0.0.1:8787/health

Override the bind address and data directory with:

MARINARA_SERVER_ADDR=0.0.0.0:8787
MARINARA_DATA_DIR=/path/to/data

Non-loopback clients fail closed unless the runtime has explicit access control. Configure BASIC_AUTH_USER and BASIC_AUTH_PASS, IP_ALLOWLIST, or an intentional compatibility opt-in such as ALLOW_UNAUTHENTICATED_PRIVATE_NETWORK=true for trusted LAN/private-network access. Set CORS_ORIGINS or CSRF_TRUSTED_ORIGINS if your desktop client origin is not one of the runtime defaults.

Remote-capable features must use the shared API HTTP pipeline: src/shared/api wrapper, remote-runtime.ts allowlist, /api/invoke or a dedicated HTTP route, then src-tauri/src/http_dispatch.rs and the focused Rust module. Do not call the server directly from feature or engine code.

Run The Remote Runtime With Docker

Build and start the Rust API server with Docker Compose:

docker compose up --build

The compose service exposes:

http://127.0.0.1:8787

To build directly from GitHub, use Docker's remote git context:

docker build -t marinara-engine-server https://github.com/Pasta-Devs/Marinara-Engine.git#refactor
docker run --rm -p 8787:8787 -v marinara-server-data:/data marinara-engine-server

Command Use it when

pnpm test:ui Build the frontend, start a fresh preview server on port 4175 by default, and run Playwright browser smoke tests. Set PLAYWRIGHT_PORT if that port is occupied.
pnpm test:ui:run Start a fresh preview server on the Playwright port and run Playwright only after a build already exists.
cargo check --manifest-path src-tauri/Cargo.toml --workspace Rust commands, capability crates, provider transport, storage, assets, or integrations changed.
pnpm check:docs Docs, agent instructions, or repo skills changed.
pnpm check Run the standard TypeScript, Rust, and docs checks together.

How Tauri Uses The Frontend

sequenceDiagram
  participant Dev as Developer
  participant Tauri as pnpm tauri dev
  participant Vite as Vite dev server :1420
  participant Rust as Rust Tauri host
  participant Remote as marinara-server HTTP runtime
  participant UI as React webview

  Dev->>Tauri: pnpm tauri dev
  Tauri->>Vite: beforeDevCommand pnpm dev
  Tauri->>Rust: compile and launch
  Rust->>UI: create desktop window
  UI->>Vite: load app from devUrl
  UI->>Rust: invoke embedded commands by default
  UI->>Remote: route supported calls over HTTP when Remote Runtime URL is set

Run These Docs

Serve the static docs locally:

pnpm docs:dev

Then open:

http://127.0.0.1:4174/

You can also open docs/developer/index.html directly from the filesystem.

Use pnpm docs:dev, not pnpm docs. The shorter command collides with pnpm/npm package documentation behavior.

Keep run/build command changes in sync with package.json, vite.config.ts, and src-tauri/tauri.conf.json.
