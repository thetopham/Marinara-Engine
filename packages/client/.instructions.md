## Architecture Rules

### State Management Division

| What                                                              | Where                                        | Persist?           |
| ----------------------------------------------------------------- | -------------------------------------------- | ------------------ |
| UI preferences (theme mode, fonts, sidebar, appearance)           | `ui.store.ts` (Zustand)                      | Yes (localStorage) |
| Server data (characters, chats, messages, presets, synced themes) | React Query hooks                            | No (cache)         |
| Chat runtime (active chat, streaming, drafts)                     | `chat.store.ts` (Zustand)                    | No                 |
| Agent execution (thoughts, debug, echo)                           | `agent.store.ts` (Zustand)                   | No                 |
| Game state (scene, combat, encounters)                            | `game-state.store.ts` / `encounter.store.ts` | No                 |
| Component-local toggles                                           | `useState`                                   | No                 |

**Rules:**

- Async server calls belong in **hooks** (React Query mutations), not in Zustand stores.
- Zustand stores are synchronous state + actions only.
- Use `useShallow()` from `zustand/react/shallow` when selecting multiple fields.
- Only `ui.store.ts` uses `persist` middleware.

### Navigation Model

Navigation is **state-based, not URL-based**. There is no URL router.

- **Editor routing:** Setting `characterDetailId`, `lorebookDetailId`, `presetDetailId`, etc. in `ui.store.ts` causes `AppShell` to render the corresponding editor in place of `ChatArea`.
- **Panel routing:** Setting `rightPanel` in `ui.store.ts` switches the right panel content.
- **Modal routing:** Setting `modal: { type, props? }` in `ui.store.ts` triggers `ModalRenderer`.

```typescript
// Open a character editor
useUIStore.getState().openCharacterDetail(characterId);

// Open a panel
useUIStore.getState().openRightPanel("lorebooks");

// Open a modal
useUIStore.getState().openModal("create-character");
useUIStore.getState().openModal("edit-agent", { agent: agentData });
```

### Code Splitting

All heavy editors are lazy-loaded in `AppShell.tsx`:

```typescript
const CharacterEditor = lazy(() =>
  import("../characters/CharacterEditor").then((m) => ({ default: m.CharacterEditor })),
);
```

**When adding a new editor or heavy component**, always lazy-load it in `AppShell` with `Suspense`.

### Component Pattern

```typescript
// Standard component structure
interface Props {
  someId: string;
  onClose: () => void;
}

export function MyComponent({ someId, onClose }: Props) {
  // 1. Zustand selectors
  const setting = useUIStore((s) => s.someSetting);

  // 2. React Query hooks
  const { data, isLoading } = useSomething(someId);
  const mutation = useUpdateSomething();

  // 3. Local state
  const [editing, setEditing] = useState(false);

  // 4. Memos and callbacks
  const filtered = useMemo(() => /* ... */, [data]);
  const handleSave = useCallback(async () => {
    await mutation.mutateAsync({ /* ... */ });
  }, [mutation]);

  // 5. Render
  if (isLoading) return <Loader2 className="animate-spin" />;
  return <div>...</div>;
}
```

### React Query Conventions

- **Query key factories:** `chatKeys.list()`, `chatKeys.detail(id)`, `chatKeys.messages(chatId)`
- **Stale times:** 5 minutes for data queries, 30 seconds for streaming-adjacent, `Infinity` for static resources (fonts)
- **Enabled guards:** `enabled: !!id` for nullable ID params
- **Invalidation on mutation:** Always `onSuccess: () => qc.invalidateQueries({ queryKey: ... })`
- **Optimistic updates:** Used for message edits — cancel, snapshot, update, rollback on error

### API Client Usage

All server calls go through `@/lib/api-client`:

```typescript
import { api } from "@/lib/api-client";

// Standard CRUD
const data = await api.get<Character[]>("/characters");
const created = await api.post<Character>("/characters", body);
await api.patch(`/characters/${id}`, body);
await api.delete(`/characters/${id}`);

// File upload (multipart)
await api.upload("/avatars", formData);

// File download (triggers save dialog)
await api.download(`/characters/${id}/export`, "character.json");

// SSE streaming (all event types, preferred for generation)
for await (const event of api.streamEvents("/generate", body, abortController.signal)) {
  // event: { type: "token" | "agent_update" | "game_state" | "done" | "error", data, ... }
  if (event.type === "token") buffer += String(event.data ?? "");
}
```

**All requests use `cache: "no-store"`.** Do not add caching headers.

### Styling Rules

- Use **Tailwind utility classes** as the primary styling method.
- Theme colors use CSS custom properties: `var(--primary)`, `var(--background)`, `var(--border)`, etc.
- **Do not write inline styles** for things Tailwind can handle.
- Global styles live in `src/styles/globals.css`, organized into 20 labeled sections.
- Custom themes are injected via `<style>` tags by `CustomThemeInjector.tsx`.
- Synced custom themes come from `/api/themes`; only extensions remain browser-local.
- The app supports dark and light mode via `data-theme` on `<html>`.
- Two visual themes exist: `"default"` and `"sillytavern"`.

### Modal & Panel Conventions

**Modals:**

- All modals use `<Modal open={open} onClose={onClose} title="..." width="max-w-lg">` from `components/ui/Modal.tsx`.
- Register new modal types in `ModalRenderer.tsx`'s switch statement.
- Pass data via `openModal(type, { props })`.

**Panels:**

- Right panels are registered in `RightPanel.tsx`'s `PANEL_CONFIG` and `PANELS` maps.
- Panels use module-level `mountedPanels` Set for persistence — once mounted, never unmounted (hidden via CSS).
- Settings panels use the same persistence pattern with `mountedSettingsTabs`.

**Overlays & Popovers:**

- Use `createPortal(content, document.body)` to escape parent stacking contexts.
- Position with `useLayoutEffect` + computed coordinates.

**Row Action Buttons (Settings Panels):**

- Action buttons (delete, duplicate, set-as-default, etc.) live in an **absolutely positioned pill container** that overlays the row content.
- Container classes: `absolute right-2 top-1/2 -translate-y-1/2 flex shrink-0 items-center gap-0.5 rounded-lg bg-[var(--sidebar)] px-1 py-0.5 opacity-0 shadow-sm ring-1 ring-[var(--border)] transition-opacity group-hover:opacity-100 max-md:opacity-100`
- The parent row element must have `group relative` in its className.
- **Desktop:** hidden by default (`opacity-0`), revealed on hover (`group-hover:opacity-100`).
- **Mobile:** always visible (`max-md:opacity-100`).
- Icon size inside action buttons: `0.75rem` (consistent across all panels).
- All action buttons use `active:scale-90` for press feedback.
- Delete buttons always show the icon in `text-[var(--destructive)]` (not muted with hover-to-red).
- Duplicate / neutral action icons use `text-[var(--muted-foreground)]` as default color.
- Destructive actions (delete) must include a `confirm()` dialog.
- Do **not** add a redundant Edit button — clicking the row itself opens the detail/editor view.

### Animation Patterns

- **Sidebar slide-in/out:** Framer Motion `AnimatePresence` with spring transitions.
- **Message entrance:** `motion.div` with stagger.
- **Modals:** CSS transitions (not Framer Motion) for React StrictMode compatibility.
- **Background transitions:** `CrossfadeBackground` component for smooth image swaps.

## Shared Package Contract

All types, schemas, and constants are in `packages/shared`. The frontend imports from `@marinara-engine/shared`.

- **Schemas (Zod):** Input validation for forms — use `z.infer<typeof schema>` for types.
- **Types:** Interfaces for API responses and store shapes.
- **Constants:** `APP_VERSION`, `DEFAULT_GENERATION_PARAMS`, `PROVIDERS`, `CHAT_MODES`, model lists.
- **Utilities:** `resolveMacros()` for template variables, `wrapInXml()` for XML wrapping.

**When adding a new entity or modifying data shapes, update `packages/shared` first**, then update hooks and components.

## How to Add Common Things

### New Entity (e.g., "Playlists")

1. Add Zod schemas + types in `packages/shared/src/schemas/` and `packages/shared/src/types/`.
2. Export from `packages/shared/src/index.ts`.
3. Add server routes in `packages/server/src/routes/`.
4. Create a hook file: `packages/client/src/hooks/use-playlists.ts` following the React Query pattern.
5. Create a panel: `packages/client/src/components/panels/PlaylistsPanel.tsx`.
6. Register the panel in `RightPanel.tsx` (`PANEL_CONFIG` + `PANELS`).
7. Add a toggle button in `TopBar.tsx`.
8. Create an editor: `packages/client/src/components/characters/PlaylistEditor.tsx` (lazy-loaded in `AppShell`).
9. Add UI store fields: `playlistDetailId`, `openPlaylistDetail()`, `closePlaylistDetail()`.

### New Modal

1. Create the modal component in `components/modals/`.
2. Accept `{ open, onClose }` props, wrap content in `<Modal>`.
3. Add the type string to the switch in `ModalRenderer.tsx`.
4. Open with `openModal("your-type", { optionalProps })`.

### New Setting

1. Add the field + setter to `UIState` interface in `ui.store.ts`.
2. Initialize the default value in the store creator.
3. Add the control in the appropriate settings subtab component.
4. Use `ToggleSetting` from `SettingControls.tsx` for booleans.

### New Hook

1. Create `use-your-feature.ts` in `hooks/`.
2. Define query key factory at the top of the file.
3. Export individual hooks: `useThings()`, `useThing(id)`, `useCreateThing()`, `useUpdateThing()`, `useDeleteThing()`.
4. Follow the established mutation → invalidation pattern.

## Common Mistakes to Avoid

- **Don't create index.ts barrel files.** This project uses direct imports everywhere.
- **Don't use URL-based routing.** Navigation is state-based via `ui.store.ts`.
- **Don't put async logic in Zustand stores.** Use React Query mutations in hooks.
- **Don't forget to lazy-load new editors** in `AppShell.tsx`.
- **Don't add React Query `refetchOnWindowFocus`.** It's disabled globally in `main.tsx`.
- **Don't use `any` without good reason** — but `@typescript-eslint/no-explicit-any` is OFF, so it compiles either way.
- **Don't modify `packages/shared` without checking server impact.** It's a shared contract.
- **Don't use `fetch()` directly.** Always use `api.*` from `@/lib/api-client`.
